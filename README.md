# Proyecto Lucky Air - Sitio Web Optimizado para SEO

Este proyecto es una página web completa y optimizada para motores de búsqueda para Lucky Air, una aerolínea ficticia especializada en vuelos a destinos chinos. El sitio incluye funcionalidades avanzadas de búsqueda de vuelos, ofertas promocionales, y está completamente optimizado con meta tags, Open Graph, Twitter Cards y datos estructurados Schema.org.

## Propósito del sitio

Lucky Air es una landing page moderna de aerolínea digital enfocada en viajes a China. El sitio ofrece:

- **Buscador avanzado de vuelos** con selección de tipo de viaje, aeropuertos de origen/destino, fechas y cantidad de pasajeros
- **13 ofertas especiales** de viajes a destinos chinos con precios en USD y CNY
- **Servicios completos** de aerolínea digital con autoservicio
- **Información corporativa** sobre Lucky Air y su modelo de negocio
- **Formulario de contacto** y enlaces a redes sociales
- **Optimización SEO completa** para mejor visibilidad en motores de búsqueda

## Características técnicas implementadas

### Optimización SEO
- Meta tags descriptivos con palabras clave estratégicas ("vuelos China", "Lucky Air", "vuelos baratos")
- Open Graph y Twitter Cards para compartir en redes sociales
- Schema.org JSON-LD para datos estructurados de aerolínea
- Alt texts optimizados en todas las imágenes con keywords relevantes
- URLs canónicas y meta robots configurados
- Estructura HTML5 semántica para mejor indexación

### Diseño y funcionalidad
- Diseño responsive que se adapta a móviles y escritorio
- Carrusel de imágenes interactivo con navegación por puntos
- Formulario de búsqueda de vuelos con validaciones
- Grid de ofertas con imágenes, rutas, fechas y precios
- Sección de servicios con iconos y descripciones
- Navegación suave con anclas internas

## Instrucciones de uso

### Visualización local

1. Abrir la carpeta del proyecto en el explorador de archivos
2. Hacer doble clic en `index.html` para abrir en el navegador
3. Verificar que todas las secciones carguen correctamente
4. Probar la responsividad cambiando el tamaño de la ventana

### Publicación en GitHub Pages

1. Subir todos los archivos al repositorio de GitHub
2. Ir a Settings → Pages
3. Seleccionar la rama principal y carpeta raíz (/)
4. El sitio estará disponible en `https://usuario.github.io/repositorio`

### Verificación SEO

Para verificar la optimización SEO:
- Usar herramientas como Google Search Console
- Validar Schema.org en https://validator.schema.org
- Probar Open Graph en https://opengraph.xyz
- Verificar meta tags con inspectores web
